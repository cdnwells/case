# Case Project Structure Analysis

## Overview
Case is an AI-powered mobile assistant system inspired by the CASE robot from Interstellar. The system consists of a React Native mobile app, a Node.js hub server, and multiple Python worker services for different AI/automation tasks.

## Architecture

```
┌─────────────────┐
│  Android App    │ (React Native + Expo)
│  (Port: Client) │
└────────┬────────┘
         │ HTTP
         ▼
┌─────────────────┐
│  Hub        │ (Node.js + Fastify)
│  (Port: 5000)   │ Reverse Proxy + API Key Management
└────────┬────────┘
         │ Forwards requests
         ▼
┌─────────────────────────────────────────────┐
│          Python Workers (FastAPI)            │
├─────────────────┬───────────────┬───────────┤
│  GPT Worker     │  SSH Worker   │  Ollama   │
│  (Port: 8000)   │  (Port: ???)  │  Worker   │
│  Chat + OpenAI  │  Shell Exec   │  (Local)  │
└─────────────────┴───────────────┴───────────┘
```

## 2. Hub (`/hub/`)

### Tech Stack
- **Framework**: Fastify (Node.js)
- **Language**: JavaScript (ES Modules)
- **Purpose**: Reverse proxy + API key management

### Architecture
**File**: `hub.js` (120 lines)

The hub acts as a single entry point for the Android app:
1. Receives requests from mobile app
2. Adds authentication headers (X-API-key)
3. Forwards to appropriate Python worker
4. Handles timeouts and error responses

### Configuration
```javascript
PORT: 5000 (default)
PYTHON_WORKER_URL: http://localhost:8000
FORWARD_TIMEOUT_MS: 120000 (2 minutes)
HUB_API_KEY: (from .env)
LOG_LEVEL: info
```

### Endpoints
- **GET /health**: Health check (not forwarded)
- **ALL /***: Catch-all reverse proxy to Python workers

### Security Features
- API key injection to worker requests
- Header sanitization (removes host, connection, keep-alive)
- Timeout protection (504 Hub Timeout)
- Error handling (502 Bad Hub)

## 3. Workers (`/workers/`)

### 3.1 GPT Worker (`/workers/gpt/`)

**Purpose**: OpenAI GPT integration with shell command extraction

#### Tech Stack
- **Framework**: FastAPI (Python)
- **AI**: OpenAI GPT-4o
- **Config**: Pydantic Settings

#### Structure
```
workers/gpt/
├── app/
│   ├── api/
│   │   ├── dependencies.py      # Auth middleware
│   │   └── routes/
│   │       └── chat.py          # Chat endpoint
│   ├── auth/                    # OAuth/API key auth
│   ├── core/                    # Core utilities
│   ├── models/                  # Pydantic models
│   ├── services/
│   │   ├── openai_service.py
│   │   ├── message_parser.py    # Shell command extraction
│   │   └── hub_service.py   # Send commands to hub
│   └── config.py                # Settings
├── docs/
│   └── system_prompt.md         # Case personality definition
├── specs/
│   ├── 001-gpt-worker/
│   ├── 002-ssh-worker/
│   └── 003-shell-command-extraction/
└── main.py
```

#### Features
- **Chat Processing**: Receives user messages, sends to GPT-4o
- **Command Extraction**: Parses response for JSON shell commands
- **Hub Communication**: Sends extracted commands to hub
- **Case Personality**: Dry, efficient humor (75%) inspired by Interstellar

#### Configuration (`config.py`)
```python
HOST: 0.0.0.0
PORT: 8000
OPENAI_API_KEY: (required)
OPENAI_MODEL: gpt-4o
OPENAI_MAX_TOKENS: 4096
OPENAI_TEMPERATURE: 0.7
API_KEY: (for hub auth)
HUB_COMMAND_URL: (optional)
ALLOWED_ORIGINS: *
RATE_LIMIT: 100 requests/60s
```

#### System Prompt (Case Personality)
Located at: `workers/gpt/docs/system_prompt.md`

**Key Rules**:
1. Never refer to self as "assistant" or "AI"
2. Speak Korean
3. Be concise, no corporate pleasantries
4. Humor setting: 75%
5. Return shell commands as JSON blocks:
```json
{
  "action": "execute",
  "command": "ls -la",
  "confirm": true,
  "description": "List files"
}
```

#### API Endpoints
**POST /chat**
- Request: `{ content: string }`
- Response: `{ message: { content, role, status, parsed_content } }`
- Auth: API key or OAuth

## Data Flow

### Typical Message Flow
1. **User Input**: User types message in Android app
2. **ChatInput Component**: Validates and sends via API service
3. **Hub**: Receives at `:5000`, adds API key, forwards to GPT worker
4. **GPT Worker**:
   - Authenticates request
   - Sends message to OpenAI GPT-4o with Case personality
   - Receives response
   - Parses for shell commands (JSON blocks)
   - Sends commands to hub (if any)
5. **Command Execution**: Hub forwards commands to SSH worker
6. **Response**: Cleaned text (without command blocks) returned to app

### Command Extraction Example
**GPT Response**:
```
파일 목록을 확인해드리겠습니다.

```json
{
  "action": "execute",
  "command": "ls -la",
  "confirm": true,
  "description": "List directory contents"
}
```

완료했습니다.
```

**Parsed Result**:
- `text`: "파일 목록을 확인해드리겠습니다.\n\n완료했습니다."
- `commands`: `[{ action: "execute", command: "ls -la", ... }]`

## Security

### Authentication Layers
1. **Mobile App**: Biometric authentication (Face ID/Touch ID)
2. **Hub**: No auth (trusted internal network)
3. **Workers**: API key (injected by hub) OR OAuth token

### Security Features
- API key validation
- OAuth introspection support
- Rate limiting (100 req/60s)
- CORS configuration
- Command confirmation flags (prevent destructive operations)

## Development Workflow

### Starting the System
```bash
# Terminal 1: Hub
cd hub
npm start  # or node hub.js

# Terminal 2: GPT Worker
cd workers/gpt
source venv/bin/activate
python main.py

# Terminal 3: SSH Worker
cd workers/ssh
source venv/bin/activate
python main.py

# Terminal 4: Android App
cd android
npm start
```

### Testing
```bash
# Health check
curl http://localhost:5000/health

# Chat endpoint
curl -X POST http://localhost:5000/chat \
  -H "Content-Type: application/json" \
  -d '{"content": "안녕하세요"}'
```
